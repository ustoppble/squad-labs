// questions

Frequently asked questions

Overclock is a multi-agent IDE and coding cockpit. It lets you orchestrate many AI coding agents in parallel, each running in its own visible pane. A pilot agent reads your context, breaks down tasks, and delegates to specialists — scout, builder, reviewer — then synthesizes the results. You see every step, in real time.
Overclock supports Claude Opus 4.8, Claude Sonnet 4.6, Claude Haiku 4.5, Gemini 2.0 Flash, GPT-4o, and Codex — with more being added. You can route different agents to different models and mix them freely within the same workspace.
Cursor and Copilot are single-agent tools: one model, one conversation, one pane. Overclock is a multi-agent orchestration layer. Tasks fan out to specialized workers running concurrently — visible, inspectable, and controllable. The difference is like comparing a single developer to a coordinated engineering team.
Overclock requires internet connectivity to reach model APIs. The cockpit UI, workspace layout, and task tracking work offline, but agents need API access to function. Self-hosted model support via Ollama is on the roadmap.
Install the CLI with npm i -g @overclock/cli, run overclock init in your project, and spawn your first agent squad with overclock run. The desktop app and VS Code extension are available for those who prefer a GUI.
