10 Áreas Funcionais

Cada área é uma camada do ADE. Juntas formam o ambiente agêntico completo — do pane ao orquestrador, do arquivo à voz.

01

Panes

Multi-chat nativo. Cada pane é um agente isolado com modelo, provedor e contexto próprio. Layout livre, sem abas.

02

Squad

Orquestrador canônico. Delegation gate define quem faz o quê. Handoff event-driven entre workers com escalation automático.

03

Catalog

Marketplace de squads, skills e configs prontos. Instalar em 1 clique. Compartilhar receitas entre projetos.

04

Providers

Todos os provedores num painel: Anthropic, OpenAI, Google, Groq, Ollama, OpenRouter. Trocar modelo por pane.

05

MCP Tools

35 tools in-process via servidor MCP embutido. Filesystem, shell, browser, git, imagem, vídeo, áudio, pesquisa.

06

Mission Room

Cria, rastreia e faz handoff de missões entre agentes. Log em tempo real. Reabre sessão exatamente onde parou.

07

Files

Workspace tabs, file tree integrado, session picker, drag-and-drop de contexto. Nunca perde o fio da meada.

08Pro

ClockVoice

Push-to-talk direto no ADE. STT local (Whisper), Groq ou qualquer provider OpenAI-compat. Pro only.

09

Settings

Controle granular: tema, atalhos, custo por modelo, limite de tokens, auto-save, notificações.

10

Auth

Autenticação segura, licença por máquina, gerenciamento de plano. Tudo local — sem dados de código na nuvem.

Como o ADE Funciona

01

Missão

Você descreve o objetivo no Mission Room. O orquestrador quebra em tarefas.

02

Squad

Workers especializados recebem tarefas via delegation gate. Cada um opera num pane isolado.

03

Execução

35 MCP tools disponíveis: ler arquivo, rodar shell, buscar, gerar imagem, controlar browser.

04

Handoff

Resultado vai de volta ao orquestrador. Aprovação humana quando necessário. Missão fechada.

Por que ADE, não IDE?

ADE vs IDE: o que muda na prática

IDE (passado)

Você escreve código. A IA sugere na margem. Uma janela, um agente, você no volante.

Overclock ADE

Agentes executam. Você define objetivos, aprova decisões, revisa artefatos. O orquestrador coordena quem faz o quê.

O que muda

Contexto não se perde entre agentes. Custo visível por token. Handoff event-driven. Squad configurável por projeto.
