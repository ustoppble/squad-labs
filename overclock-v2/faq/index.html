FAQ

Perguntas Frequentes

As 12 perguntas que todo dev faz antes de testar o ADE.

Produto

O que é um ADE (Agentic Development Environment)?
Um ADE é um ambiente de desenvolvimento projetado para orquestração de múltiplos agentes de IA — não para humanos escreverem código linha a linha. No Overclock ADE, você define missões; agentes (Claude, Codex, Gemini e outros) executam em paralelo, coordenados por um orquestrador canônico com delegation gate e handoff event-driven. É a evolução do IDE para a era agêntica.
O que é o orquestrador canônico e o delegation gate?
O orquestrador é um agente que recebe a missão, a quebra em tarefas com dependências, e distribui para workers via delegation gate. O delegation gate — a camada que decide qual agente faz o quê, como um gerente de projetos automático — escolhe o worker certo baseado em especialização, modelo, contexto e custo estimado. Workers reportam via handoff event-driven; se bloqueiam, o orquestrador escalona para você.
O que é o Mission Room?
Mission Room é o centro de comando de missões do Overclock ADE. Você cria missões com objetivo e critérios, acompanha o status de cada tarefa em tempo real, vê o log de decisões do orquestrador, e faz handoff manual quando quiser. A missão persiste entre sessões — feche o Overclock e reabra onde parou.
Qual a diferença entre Overclock ADE e um IDE como Cursor ou Windsurf?
IDEs (Cursor, Windsurf, VS Code) foram projetados para humanos escreverem código, com IA como copiloto. Overclock ADE é projetado para agentes executarem objetivos — múltiplos agentes em paralelo, orquestrador canônico, delegation gate, 35 MCP tools in-process. Overclock não tem editor de código nativo por design: use seu editor preferido para edição pontual; use Overclock para orquestrar missões maiores.Ver tabela comparativa →

Técnico

O que são MCP tools e quantas o Overclock tem?
MCP (Model Context Protocol) é um protocolo que permite agentes de IA usarem ferramentas externas — ler arquivos, rodar shell, buscar na web, gerar imagem, etc. Overclock embute um servidor MCP in-process com 35 tools: filesystem, shell, browser (Playwright), git, geração de imagem/vídeo/áudio, pesquisa web, memória persistente, Supabase e mais. In-process significa sem latência de rede — as tools rodam no mesmo processo que o Overclock.
Quais provedores de IA o Overclock suporta?
Anthropic (Claude Haiku, Sonnet, Opus), OpenAI (GPT-4o, GPT-4, Codex), Google (Gemini Pro, Gemini Flash), Groq (Llama, Mixtral — inferência rápida), Ollama (modelos locais como Llama 3, Mistral, Qwen), e OpenRouter (aggregator de modelos). Você pode ter panes com providers diferentes abertos ao mesmo tempo.
Posso usar modelos locais (Ollama) no Overclock?
Sim. Ollama é um dos providers suportados. Configure o endpoint local (geralmente http://localhost:11434) nas Settings, e use modelos como Llama 3.2, Mistral, Qwen e outros direto nos panes — sem custo de token de API.
O Overclock funciona offline?
O Overclock (app) funciona offline. O que precisa de conexão são os LLMs cloud (Claude, GPT, Gemini, Groq) — mas se usar Ollama com modelos locais, você pode orquestrar missões completamente offline. Filesystem, shell, git e outras MCP tools locais também funcionam offline.
Em quais sistemas operacionais o Overclock funciona?
macOS, Windows e Linux. O Overclock é construído com Tauri — um runtime nativo leve que não requer Electron ou Chrome embutido.

Planos

Qual a diferença entre Boost e Pro?
Boost (R$97/mês) é o ADE completo: panes ilimitados, Squad orchestration, Catalog, 6 providers, 35 MCP tools, Mission Room, Files, Settings e Auth. Pro (R$197/mês) adiciona ClockVoice (push-to-talk, STT local Whisper, Groq STT, OpenAI-compat STT). Se não precisar de voz, Boost tem tudo o que você precisa para desenvolvimento agêntico.
O que é ClockVoice? Por que é Pro only?
ClockVoice é o sistema de push-to-talk nativo do Overclock. Segure um atalho configurável, fale, e a transcrição vai direto para o pane ativo. Suporta STT local (Whisper, offline), Groq STT (cloud rápido) e qualquer provider OpenAI-compat. É Pro only porque exige processamento de áudio — infraestrutura adicional que não faz sentido em todos os planos.

Privacidade

Meu código sai para os servidores do Overclock?
Não. Overclock é local-first: seu código, contexto e histórico ficam na sua máquina. O que vai para fora é apenas o que você envia como contexto ao LLM escolhido (Anthropic, OpenAI, etc.) — o mesmo que acontece ao usar Claude.ai ou ChatGPT diretamente. Servidores Overclock nunca recebem seu código.
