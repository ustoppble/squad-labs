Arquitetura

Como o ADE Funciona

Overclock não é um IDE com plugin de IA. É um ADE — Agentic Development Environment — projetado para orquestração multi-agente desde o início. Veja como o fluxo funciona.

O Ciclo do Squad

01

Você cria uma Missão

No Mission Room, descreve o objetivo em linguagem natural. Ex: "Refatorar o módulo de auth para usar JWT stateless e escrever testes para todos os endpoints."

A missão fica registrada com status pending. Nenhum agente executa ainda.

02

Orquestrador quebra em tarefas

O orquestrador — o modelo mais capaz configurado no Squad (Claude Opus, GPT, ou qualquer provider) — lê a missão, analisa o codebase via MCP tools e gera um plano de tarefas com dependências.

Ex: Tarefa A — leitura do módulo atual. Tarefa B (depende de A) — escrever testes. Tarefa C (depende de B) — implementar JWT.

03

Delegation gate distribui

O delegation gate — a camada que decide qual agente faz o quê, como um gerente de projetos automático — escolhe o worker certo para cada tarefa, baseado em especialização, modelo, contexto e custo estimado. Workers são panes com personas configuradas.

Um worker Haiku para leitura de arquivos (barato + rápido). Um worker Sonnet para os testes. Um worker Opus para a implementação JWT.

04

Workers executam com 35 MCP tools

Cada worker tem acesso ao servidor MCP embutido (in-process, sem latência de rede). Pode ler arquivos, rodar shell, buscar docs, commitar, abrir browser — tudo nativo.

O worker de testes roda `npm test` via Bash MCP tool, lê os erros, corrige e repete até verde.

05

Handoff event-driven

Quando um worker termina, dispara um evento de handoff — o sinal de que a tarefa está pronta para o próximo passo. O orquestrador recebe, valida o resultado, e decide: aprovar (próxima tarefa), rejeitar (worker tenta de novo) ou escalar (para você).

Se o worker falha 3 vezes, a missão entra em status blocked e você recebe notificação.

06

Missão encerrada

Todas as tarefas concluídas → missão fechada. Log completo com decisões, custo por worker, tempo por etapa. Sessão persiste — feche e reabra onde parou.

O histórico fica no Mission Room. Reutilize o squad em outro projeto via Catalog.

Para desenvolvedores

Arquitetura Técnica

Tauri + Sidecar Node

Desktop app nativo (macOS, Windows, Linux) construído com Tauri. Sidecar Node roda o servidor MCP e o orquestrador in-process — sem servidor externo.

Local-First

Nenhum dado de código sai para servidores Overclock. O ADE é local-first: código, contexto e histórico ficam na sua máquina. Só o token da API do provedor vai para o LLM escolhido.

MCP In-Process

O servidor MCP (Model Context Protocol) roda dentro do mesmo processo — sem latência de rede. Agentes chamam tools com a mesma velocidade de uma chamada de função local.

Event-Driven Handoff

Workers comunicam por eventos (não polling). O orquestrador recebe handoffs assíncronos e processa as próximas tarefas sem bloquear a UI.

Isolation por Pane

Cada pane tem contexto isolado. Um worker com 128k tokens de contexto não polui o contexto de outro. Paralelismo real, sem interferência.

Provider Agnostic

O orquestrador não depende de nenhum provedor específico. Anthropic, OpenAI, Groq, Ollama, OpenRouter — troque por pane sem reconfigurar a pipeline.
